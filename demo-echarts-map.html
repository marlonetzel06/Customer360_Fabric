Azure Map Preview – Canvas 2D vs Leaflet

Wie das Theme-Farbschema auf dem Map-Visual wirkt – aktuell (Canvas 2D) vs. Vorschlag (Leaflet + OSM)

Aktuell: Canvas 2D Alt
Einschränkungen 6 grobe Kontinent-Blobs, falsche Ländergrenzen, statische Bubble-Positionen, kein Tooltip, kein Zoom — zeigt nur grob ob Farben passen.
Vorschlag: Leaflet + OpenStreetMap Neu
Vorteile Echte Satelliten-/Straßenkarte (OSM-Tiles), korrekte Koordinaten, interaktiver Tooltip, Zoom & Pan, Bubble-Größe nach Datenwert. Kein Power BI Account, kein Backend, kein GeoJSON-Download.

Integration ins Theme-Tool (Auszug)

// AzureMapPreview.jsx – Leaflet Bubble Map import { MapContainer, TileLayer, CircleMarker, Tooltip } from 'react-leaflet'; function AzureMapPreview({ theme }) { const colors = theme.dataColors; return ( <MapContainer center={[20, 10]} zoom={2} style={{ height: '360px' }}> <TileLayer url="https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png" /> {CITIES.map((city, i) => ( <CircleMarker key={city.name} center={[city.lat, city.lon]} radius={Math.sqrt(city.value) * 2.5} fillColor={colors[i % colors.length]} color="white" fillOpacity={0.85} weight={1.5} > <Tooltip>{city.name}: {city.value}</Tooltip> </CircleMarker> ))} </MapContainer> ); } // Theme-Änderung: colors-Prop updaten → React re-rendert → neue Farben // Kein applyTheme() API nötig, kein PBI-Token, kein Backend
