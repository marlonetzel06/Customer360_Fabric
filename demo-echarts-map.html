Azure Map Preview – Canvas 2D vs Apache ECharts

Wie das Theme-Farbschema auf dem Map-Visual wirkt – aktuell (Canvas) vs. Vorschlag (ECharts)

Aktuell: Canvas 2D Alt
Einschränkungen Vereinfachte Kontinent-Polygone (6 Formen), keine echten Ländergrenzen, statische Bubble-Positionen, kein Hover/Tooltip, kein Zoom. Zeigt nur grob ob Farben "passen" – nicht wie das Visual wirklich aussieht.
Neu: Apache ECharts Vorschlag
Vorteile Echte Ländergrenzen (GeoJSON), interaktiver Tooltip, Zoom & Pan, live Theme-Updates ohne Reload, skaliert auf beliebige Größen. Kein Power BI Account nötig, keine Azure-Abhängigkeit.

Integration ins Theme-Tool (Auszug)

// AzureMapPreview.jsx – ECharts Bubble Map import ReactECharts from 'echarts-for-react'; import worldJson from './world.json'; // ~200KB, einmalig gebundlet echarts.registerMap('world', worldJson); function AzureMapPreview({ theme }) { const colors = theme.dataColors; const option = { backgroundColor: theme.background ?? '#fff', geo: { map: 'world', roam: true, // Zoom + Pan itemStyle: { areaColor: '#cce8f7', borderColor: '#fff', borderWidth: 0.5, }, emphasis: { itemStyle: { areaColor: colors[1] } }, }, series: [{ type: 'scatter', coordinateSystem: 'geo', data: SAMPLE_CITIES, // [{name, value:[lon,lat,size]}] symbolSize: (val) => Math.sqrt(val[2]) * 8, itemStyle: { color: (params) => colors[params.dataIndex % colors.length], opacity: 0.85, }, tooltip: { formatter: (p) => `${p.name}: ${p.value[2]}` }, }], }; // Bei Theme-Änderung: eChart.setOption(newOption) – kein Re-Mount nötig return <ReactECharts option={option} style={{ height: '360px' }} />; }
